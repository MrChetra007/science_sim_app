Physics Shot (Wave Lab) - Phase 7: Experience & Monetization
This phase focuses on the "Experience Layer" and "Monetization", adding polish, audio, and the IAP framework.

Proposed Changes
1. Audio Tone Generator (flutter_soloud)
Goal: Generate a real-time sine wave tone matching the simulation frequency.
Logic: Initialize SoLoud in Main, update frequency based on WaveState.
2. Slow Motion Mode
Goal: Allow users to see wave propagation in detail.
Logic: Add timeScale to WaveState (default 1.0, slow mo 0.1). Multiply deltaTime by timeScale in WaveNotifier.
3. Home Screen Animation
Goal: A beautiful landing screen with a full-screen wave animation.
Implementation: Create HomeScreen with a custom painter background. Use GoRouter for navigation.
4. Challenge Mode (Preview)
Goal: Introduction of the challenge system.
Implementation: A simple screen where the user must match a target frequency.
5. In-App Purchase (IAP) Service
Goal: Framework for Pro features.
Implementation: Create IapService using in_app_purchase. Setup shared_preferences for local status.

User Review Required
- Should the audio tone be enabled by default? It can be jarring for some users.
- Slow motion button placement: I suggest adding it next to the Play/Pause button in the ControlPanel.

Verification Plan
- Verify audio frequency matches visual frequency. 
- Ensure slow motion correctly scales the animation speed.
- Test IAP framework handles "pending" or "cancelled" states.
